# AI Media Content System — Product Description

## What this system is

An AI-powered content production pipeline that transforms a text idea into a
complete multimedia content package: script, images, and video.

The system is not a chatbot. It is a production pipeline — a sequence of typed,
auditable workflow stages where each stage has a defined input contract, a defined
output contract, and a clear success criterion.

The operator (human) interacts through a simple interface. All real work —
research, scripting, media planning, generation, quality review — happens inside
the pipeline, not inside the chat.

## What it produces

Given a single text idea (e.g. "a short video about specialty coffee"), the system
produces a content package containing:

- A normalized content brief
- A grounded research pack with cited sources
- Multiple content angle options
- A structured script with scenes
- A media plan with visual descriptions for each scene
- Generated images for each scene (Milestone 3+)
- An assembled video (Milestone 4+)
- A QA review report
- A complete run log with all artifacts and costs

## What it is not

- Not a chatbot or conversational assistant
- Not a single "smart prompt" that does everything
- Not a tool that publishes content automatically
- Not a replacement for human judgment on final output

## Who uses it

**Operator** — the human who submits the initial idea and approves the final output.
The operator interacts through the UI or CLI. The operator sees run status, can
review intermediate artifacts, and must explicitly approve before any output is
finalized or published.

## Success criteria

A run is considered successful when:

1. All pipeline stages complete without schema validation errors
2. Every factual claim in the research pack has a cited source
3. The script aligns with the content brief and chosen angle
4. The review report returns `verdict: approved`
5. All artifacts are stored in `runs/artifacts/{run_id}/`
6. The full run log exists in `runs/logs/run_{id}.json`
7. The run is reproducible: the same inputs produce a structurally equivalent output

## Milestone roadmap

| Milestone | Output |
|-----------|--------|
| M1 — Text pipeline | Brief + Research + Script + Review report |
| M2 — Media prompts | M1 output + MediaPlan with prompts per scene |
| M3 — Image generation | M2 output + generated images per scene |
| M4 — Video generation | M3 output + assembled video |

## Key constraints

- Media (images, video) is always generated by external APIs — never by the language model directly
- No artifact is ever overwritten — each run produces a new `run_id`
- No output is published without an explicit human approval step
- All costs are tracked per run in the RunLog
