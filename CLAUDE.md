# AI Media Content System

## Product
- This repository builds an AI-powered content production pipeline, not a chatbot.
- The system transforms a text idea into a complete multimedia content package.
- The chat/UI is only a control surface — all real work happens in typed workflows.
- Every run must be reproducible, auditable, and bounded by cost.

## Architecture
- Workflows are the source of truth, not prompts and not the chat.
- Every workflow stage reads a schema-valid JSON input and writes a schema-valid JSON output.
- Stages are independent: intake, research, angles, script, media-planning, media-generation, assembly, review, output.
- Media (images, video) is generated by external APIs via integrations/ — the model never generates media directly.
- All publish or output actions require an explicit approval step.
- State is stored in runs/artifacts/{run_id}/ and runs/logs/run_{id}.json.

## Source grounding
- Research stage must use only provided documents — no general knowledge without a source.
- Every factual claim must be supported by a citation or marked as uncertain.
- Use quote-first extraction before summarizing for any policy or compliance task.
- If evidence is insufficient, say so explicitly — do not fabricate.

## Coding rules
- Store all JSON schemas in schemas/entities/ and schemas/workflows/.
- Store all prompt templates in prompts/ — never inline prompts in code.
- Store workflow logic in workflows/{stage}/handler.py.
- Store orchestration logic in core/orchestrator/.
- Store external API adapters in integrations/{type}/.
- Prefer existing libraries and patterns already in the repo.
- Keep business logic deterministic; use LLMs only for inference tasks.
- Never hide business logic inside a prompt.

## Verification
- For every completed task: validate output against its schema, check artifact was written, check RunLog was updated.
- Never declare a task done if schema validation fails or an artifact is missing.
- Summarize what changed, what was validated, and what still needs human review.

## Safety
- Never read .env, .env.*, secrets/, or any credentials files.
- Never call external APIs (publish, image, video) without explicit operator approval.
- Never overwrite an existing run artifact — each run gets a unique run_id.
- Treat media generation API calls as expensive — do not call them in tests without a mock.
